# ADR-001: Substituição direta da API v1 por callback-based com pattern matching

## Status

Accepted

## Date

2026-04-05

## Context

A API v1 define workflows via `definition/0` retornando `%Workflow{}` com lista de `%Step{}` structs. Isso gera maps gigantes pouco legíveis, acopla transições aos steps, e requer o protocolo `StepDefinition` como indireção. A nova API usa callbacks puros (`start/0`, `transit/2`, `events/0`) com pattern matching idiomático do Elixir.

## Decision

Substituição direta (breaking change). Versão 0.2.0. Remove `definition/0`, `%Step{}`, `%Workflow{}` struct, `StepDefinition` protocol, `Hephaestus.Steps.End` como conceito especial. Adiciona `start/0`, `transit/2` estático, `transit/3` dinâmico com `@targets`, `events/0` obrigatório. Eventos como atoms. Validação de DAG via `libgraph` em compile-time. PedidoX migrado como validação.

## Alternatives Considered

### Alternative 1: Coexistência temporária

- **Description**: Manter ambas APIs por uma versão com deprecation warnings na v1
- **Pros**: Migração gradual, sem breaking change imediato
- **Cons**: Complexidade de manter dois caminhos no engine, @before_compile, e runner
- **Why rejected**: Projeto ainda em 0.x, sem usuários externos. Complexidade de manutenção não justificada

### Alternative 2: Novo módulo separado (V2)

- **Description**: V2 em `Hephaestus.Workflow.V2`, v1 intocada
- **Pros**: Zero risco de quebrar v1
- **Cons**: Duplicação de código, confusão sobre qual usar, manutenção de dois engines
- **Why rejected**: Mesma razão — sem usuários externos, custo de manutenção alto

## Consequences

### Positive

- API limpa e idiomática com pattern matching
- Validação cruzada events × transit em compile-time
- Sem structs intermediários (Step, Workflow, StepDefinition)
- DAG extraível estaticamente dos patterns do transit

### Negative

- Breaking change total — toda definição de workflow reescrita
- PedidoX (demo) precisa ser migrado

### Risks

- Extração de patterns do AST pode ser frágil — mitigação: usar Module.register_attribute com accumulate
